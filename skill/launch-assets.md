# Output Artifacts

## Standard launch brief

```text
Product:
One-line description:
Who it is for:
Problem it solves:

Why now:
- ...

Why Solana:
- ...

Core proof points:
- ...

Differentiators:
- ...

Risks or weak spots:
- ...

Demo flow:
- step 1
- step 2
- step 3

FAQ:
- ...
```

## Short form

Use when the user wants a compact artifact:

- one-line positioning
- three proof points
- one ideal user
- one demo path

## Outreach mode

When the brief is meant for judges, partners, or intros:

- keep the narrative under control
- do not oversell roadmap as current state
- highlight one concrete reason to care now
