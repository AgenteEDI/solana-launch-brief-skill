# Briefing Workflow

## Source material to inspect

- repository and recent commits
- docs and onboarding
- product site and screenshots
- X profile or launch posts
- competitor or adjacent products

## Questions to answer

### 1. Product core

- what does it actually do
- what user problem does it reduce
- what is the narrowest truthful description

### 2. User

- who feels the pain first
- what are they doing today instead
- what existing workflow does this replace or compress

### 3. Proof

- what proof points exist already
- what concrete features support the claims
- where is the product still immature

### 4. Competitive edge

- what is actually better
- what is merely different
- what should not be over-claimed

## Output behavior

- prefer sharp language over maximal language
- remove generic AI/crypto filler
- keep claims proportional to evidence
