# Solana Launch Brief

Use this skill when the user needs to explain a Solana product clearly to builders, partners, judges, or early users.

## What This Skill Is For

- founder launch briefs
- demo preparation
- grant and accelerator product framing
- ecosystem intros
- positioning cleanup
- turning repo-plus-docs into a usable story

## Default Operating Procedure

1. Read the repo, docs, landing pages, public comms, and any demo material.
2. Use [briefing.md](briefing.md) to structure the product read.
3. Use [messaging.md](messaging.md) to sharpen positioning.
4. Use [launch-assets.md](launch-assets.md) to package the final artifacts.

## Decision Standard

The output should help answer:

- what this product is
- who it is for
- why it matters now
- why it is better or different
- how to explain it without fluff

## Read These Files When Needed

- [briefing.md](briefing.md): source-to-brief workflow
- [messaging.md](messaging.md): positioning rules and tone
- [launch-assets.md](launch-assets.md): deliverable templates
