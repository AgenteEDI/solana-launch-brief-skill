# Solana Launch Brief Skill

A skill for turning a Solana project's repo, docs, product surface, and market context into a crisp founder-grade launch brief.

## What problem it solves

Many teams ship faster than they explain. They have:

- a repo
- scattered docs
- partial product pages
- raw features

But they do not have a clean operating brief for launch, demos, outreach, or ecosystem conversations.

This skill helps coding agents turn raw project context into a structured launch narrative with proof points and usable messaging.

## Core outputs

- `product brief`
- `ICP and use-case framing`
- `positioning and differentiators`
- `demo flow`
- `FAQ and proof points`

## Example prompts

- "Turn this Solana repo into a launch brief"
- "Prepare a founder-facing product brief from these docs"
- "What is the clearest positioning for this protocol?"
- "Build a demo narrative and FAQ for this launch"
